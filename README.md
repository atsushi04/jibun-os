# Jibun OS 🗂️

**AIエージェントの「仕事場」と「記憶領域」を、あなた専用に構築するClaude Codeプラグイン**

フォルダ構成そのものをAIエージェントのOSとして設計する——「フォルダ＝OS」の考え方をベースに、ヒアリング形式であなたの志向性（仕事/生活のスコープ・業務内容・タスク管理方法・振り返り習慣）を確認し、以下を自動生成します。

- 📁 **番号付きフォルダ構造** — 不変の文脈（00_context）から流動的な制作物まで、AIが迷わない「番地」設計
- 📄 **CLAUDE.md** — AIの役割定義とディレクトリの意味を記述した中核コンテキスト
- 🧠 **memories/** — 意思決定・好み・方針を蓄積してAIが次の判断に活かす記憶の仕組み
- ⚙️ **運用スキル4種** — あなた向けにパーソナライズされた daily-schedule / daily-summary / weekly-review / monthly-review

## インストール

```
/plugin marketplace add atsushi04/jibun-os
/plugin install jibun-os@jibun-os
```

## 使い方

ワークスペースにしたいディレクトリで Claude Code を起動して：

```
/jibun-os:workspace-init
```

あとは対話に答えるだけ。既存のファイルがあるディレクトリでも、**今あるものを壊さず**にマッピング案を提示してから整理します。

セットアップ完了後は、毎朝「おはよう」と話しかければ今日の工程表が、夜に「今日やったことまとめて」と言えば日次ログが生成されます。

## 生成されるワークスペースの例

```
your-workspace/
├── CLAUDE.md            # AIの役割・フォルダの意味（中核コンテキスト）
├── DAILY.md             # 毎日の工程表・作業ログ
├── WEEKLY.md / MONTHLY.md
├── 00_context/          # 不変の文脈（プロフィール・memories/）
├── 01_strategy/         # 書き換え続ける目標・戦略
├── …                    # ヒアリング結果に応じた用途別フォルダ
├── 99_memo/             # 雑多なインボックス
└── .claude/skills/      # パーソナライズ済みの運用スキル4種
```

## 設計思想

1. **固定アドレス** — 2桁番号の若い順に「不変 → 流動」で並べ、一度決めた番地は変えない
2. **不変の文脈と進行中の戦略を分ける** — AIがどちらを軸に考えるべきか迷わない
3. **記憶と判断の蓄積** — memories/ に意思決定ログを溜め、次の判断に効かせる
4. **スキルはユーザーのもの** — 生成されたスキルは `.claude/skills/` に置かれ、自由に育てられる

## 動作要件

- Claude Code
- git（daily-summary / weekly-review がコミットログを利用。なくても動作するがgit管理を推奨）
- gh CLI（タスク管理にGitHub Issuesを選んだ場合のみ）
